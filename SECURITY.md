# 보안 정책 (Security Policy)

이 저장소는 삼성증권 영크리에이터 15기 4조의 **PB Insight — VVIP 자산관리 대시보드** 프로젝트입니다.
PB가 실제 고객 상담에 사용하는 것을 전제로 한 금융 의사결정 보조 도구로, **고객 자산
데이터 등 민감정보를 다루는 것을 전제로 설계**합니다. 보안 이슈는 일반 버그보다 높은
우선순위로 처리합니다.

## 취약점 제보 방법

보안 취약점을 발견하셨다면 **공개 이슈로 올리지 마시고** 아래 채널로 비공개 제보해 주세요.

- 이메일: **team4youngcreator@gmail.com**
- 제목에 `[SECURITY]`를 붙이고, 다음을 포함해 주세요.
  - 취약점 설명과 영향 범위
  - 재현 절차 (가능하면 단계별로)
  - 발견한 브랜치/커밋

제보를 받으면 **3일 이내에 1차 회신**하는 것을 목표로 합니다. 수정이 완료되어 배포되기
전까지는 취약점 내용을 공개하지 말아 주세요.

## 지원 범위

| 브랜치 | 지원 여부 |
| --- | --- |
| `main` (배포 버전) | ✅ |
| `develop` (개발 통합) | ✅ |
| 작업 브랜치 (`feat/*`, `fix/*` 등) | ❌ — develop 머지 후 처리 |

## 비밀 정보 관리 원칙

- **API 키·비밀번호·서비스 키는 절대 커밋하지 않습니다.** 모든 비밀 값은 `.env`로만
  관리하고, 각 하위 디렉토리(frontend, backend)의 플레이스홀더만 담긴 `.env.example`을 활용합니다.
- 외부 API 키는 코드에서 **환경변수로만** 읽습니다. 하드코딩 금지.
- 고객(사용자) 정보를 다루는 코드는 로깅 시 PII 마스킹을 적용합니다.
- 에러 메시지·로그에 키, 엔드포인트 등 비밀 값을 출력하지 않습니다.

## 운영 중인 보안 장치

- **브랜치 보호**: GitFlow(`feature → develop → main`). `main` 직접 커밋 금지,
  모든 변경은 PR + 최소 1명 리뷰 승인 후 머지.
- **AI 코드 리뷰 봇(Gemini)**: 모든 PR에 자동 리뷰 운영 중.
- **GitHub 자동 스캔**: Secret scanning(비밀 푸시 감지), Code scanning(CodeQL),
  Dependabot alerts(의존성 취약점) 활성화.

자세한 협업 규칙은 [`docs/CONTRIBUTING.md`](docs/CONTRIBUTING.md)를 참고하세요.
